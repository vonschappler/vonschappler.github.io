Custom ShoutOut (text version)

Guia de referência rápida e informações adicionais sobre este script

Streamers têm várias maneiras de apoiar ou mostrar apreço por outros streamers. Raids, Hosts, anúncios (também chamados ShoutOuts) são algumas dessas maneiras. Esse script basicamente ajuda streamers na parte Ad (de advertisement) deste suporte para outras pessoas.
Além disso, nem sempre o streamer possui moderadores ali para ajudá-lo nessa publicidade aos colegas streamers, e pensando nessa situação específica, eu venho aqui para oferecer a você uma novo script que vai ajudar com isso! A lista abaixo resume o que o Custom ShoutOut pode fazer por você (como streamer): test
  • Criar um arquivo com "mensagens personalizadas" para cada streamer que você deseja chamar na sua stream
  • Permitir os Auto-ShoutOuts para raiders e streamers salvos quando eles se juntam ao seu canal ou enviam sua primeira mensagem durante uma sessão de stream (somente streamers salvos)
  • Oferecer controle total do arquivo de streamers salvos, com base em comandos e botões da interface da interface do usuário
  • Sistema de log profundo para permitir o controle de tudo o que o script faz durante uma sessão de stream
Vamos agora aprofundar um pouco nas especificações de cada parte do script!

IMPORTANTE:

Quaisquer alterações feitas em QUALQUER campo neste script entrarão em vigor após o botão "Save Settings" (no final da interface do usuário do script) for pressionado e o script for recarregado. Também é necessário dizer que a maioria das respostas do script é enviada ao whisper do usuário, o que significa para que o usuário verifique se as alterações feitas foram bem-sucedidas ou não, ele precisará ter seu bot adicionado como amigo para obter todos os whispers, caso o usuário tenha bloqueado o recebimento de whisper vindos de estranhos no Twitch.

a) Main Settings:


Ver em tamanho real (nova aba)

Este grupo é responsável pelas configurações básicas do script. É onde você controla o comportamento básico do bot.

a.1) Auto ShoutOut saved casters?
Esse recurso habilitará ou desabilitará a opção que permite o bot fazer Auto-ShoutOuts para casters/streamers salvos no seu arquivo de casters, cuja criação será detalhada posteriormente.

a.2) ShoutOut when caster joins channel?
Se ativado, esse recurso enviará um ShoutOut personalizado (quando definido) sempre que um caster salvo se visitar seu canal. Se desativado, é necessário que o caster salvo envie qualquer coisa no seu bate-papo para que o bot envie o ShoutOut em chat. Será enviado para o chat de stream apenas um ShoutOut para cada caster salvo e apenas uma vez por sessão com esse recurso. Para ShoutOuts extras, será necessário fazê-lo via comando.

a.3) Auto ShoutOut raiders?
Esse recurso enviará um Auto-ShoutOut para raiders de seu canal com base nas configurações definidas no próximo grupo.

b) Shout Raiders Settings:


Ver em tamanho real (nova aba)

Como já foi dito, este grupo controla como o bot se comportará se a função a.3 estiver ativada. É também possível enviar um teste de raid ao script (graças ao script "Raid Notify" feito por Kruiser8. Algumas alterações foram feitas nesse código para tornar esse test de raid um pouco mais realista.

b.1) Message to use for ShoutOut raiders:
É aqui que você pode definir uma mensagem personalizada para ser enviada como Auto-ShoutOut sempre que alguém fizer um raid em seu canal. Note que para o correto funcionamento do script, alguns parâmetros DEVEM ser adicionados à sua mensagem personalizada:
  • {raider}: isso será substituído automaticamente pelo nome do raider
  • {count}: isso será substituído automaticamente pela quantidade de viewers que ele trouxe consigo
  • {url}: isso será substituído automaticamente pelo link do canal Twitch do raider
  • {game}: isso será substituído automaticamente pelo último game jogado pelo raider
Exemplo de mensagem aceita pelo script:
{raider} was playing {game} and just brought {count} viewers to our channel! Let's support them visiting {url} !
IMPORTANTE:
O script verificará a presença dos parâmetros listados acima ao tentar salvar a mensagem personalizada usada para envio de ShoutOuts aos raiders. Portanto, para que a mensagem seja aceita pelo script, lembre-se de sempre dar um espaço em branco depois de cada um dos parâmetros utilizados e adicione TODOS os parâmetros na sua mensagem!
b.2) Minimum raiders to Auto ShoutOut:
Isso especifica o número de espectadores que o invasor precisa trazer consigo para o seu canal e assim receber o ShoutOut automático.

b.3) Save raiders to caster file?
Se ativado, esse recurso adicionará uma entrada ao arquivo de casters salvos para uso posterior em outras funcionalidades do script.

b.4) Test Raider AutoSO settings:
Ao clicar neste botão, o bot enviará um raid "falso" para que seja possível testar as configurações acima definidas. Se a.3 estiver ativado, o script exibirá no seu bate-papo do Twitch a mensagem personalizada definida em b.1 e se b.3 estiver ativado, ele salvará (ou atualizará) uma entrada com os detalhes do raid no arquivo de casters.
Como já mencionado, esta parte do script foi criada com a ajuda do script já existente ("Raid Notify") feito por Kruiser8, com algumas modificações no código original para uso nesta situação de teste.

c) ShoutOut Settings:


Ver em tamanho real (nova aba)

Este grupo aqui é para gerenciar como o bot controlará as configurações dos ShoutOuts "manuais" (extras), Auto-ShoutOuts e até dar a possibilidade ShoutOuts manuais a casters que não são salvos no seu arquivo casters.

c.1) Command for manual ShoutOut:
Este campo é usado para definir o comando usado para o ShoutOuts manuais, no caso de a.1 estar desativado ou se quando quiser dar ShoutOuts extras. Este comando irá fazer os ShoutOut para casters salvos.
USO:
!caster <casterName> or !caster <@casterName>
RESPOSTA:
A resposta depende das configurações salvas para este script. De fato, o bot procurará a entrada de casterName em seu arquivo de casters e postará uma mensagem de publicidade para casterName.
c.2) Permission to use the command:
Esse campo é auto-explicativo. Apenas usuários com a permissão necessária poderão usar comando definido em c.1. Por padrão, Moderadores é a configuração básica de permissão para usar este comando, mas cabe ao usuário definir a quem deseja conferir permissão para poder usar o comando.

c.3) Default message for ShoutOuts:
Semelhante ao detalhado em b.1, esse campo é usado para definir uma mensagem de anúncio a ser postada no chat. Esta mensagem é chamada "Default Message" por um motivo a ser detalhado posteriormente neste guia. Para que esta mensagem seja aceita pelo script, ele precisa ter o seguintes parâmetros assim especificados:
  • {caster}: isso será substituído automaticamente pelo nome do caster (parâmetro necessário)
  • {url}: isso será substituído automaticamente pelo link do canal da Twitch do caster (parâmetro necessário)
  • {game}: isso será substituído automaticamente pelo último game jogado por caster (parâmetro opcional)
Exemplo de mensagem aceita pelo script:
Let's support my friend {caster} visiting their {url} ! There were playing {game} on their last stream!
IMPORTANTE:
O script verificará a presença dos parâmetros listados acima ao tentar salvar a mensagem personalizada para uso em ShoutOuts de casters. Portanto, para que a mensagem seja aceita pelo script, lembre-se de sempre dar um espaço em branco após cada um dos parâmetros utilizados e não se esqueça de adicionar TODOS os parâmetros NECESSÁRIOS em sua mensagem!
c.4) Global cooldown (in seconds):
Especifica um pequeno cooldown para evitar spam do comando, caso ele seja usado por dois ou mais usuários permitidos em um curto espaço de tempo menor que o valor definido. O valor da configuração padrão é 5 segundos e pode ir até 60 segundos (1 minuto).

c.5) Allow manual shout to unsaved casters?
A ativação desse recurso concederá permissão ao script para enviar ShoutOuts a casters não-salvos no arquivo de casters usando o comando definido em c.1.

d.4) Caster Control Settings:

Ver em tamanho real (nova aba)

Esta seção aqui é o coração deste script. Primeiro, aqui é onde você define todos os comandos necessários para gerenciar o que será adicionado ao arquivo de casters que será usado pelo script.

d.1) Command to add a new caster to AutoShoutOut:
Este é o comando que você e seus editores (sim, é definido por padrão por razões de segurança, para que apenas os editores - além de você - tenha acesso a usá-lo) adicionaram um novo caster para o seu arquivo de casters. É importante mencionar que todos os casters adicionados ao seu arquivo de casters SEMPRE serão "ShoutOut" pelo script (seguindo as configurações definidas nos campos já detalhados), a menos que especificado de forma diferente com a definição ALLOW (a ser explicada mais adiante neste guia).
USO:
!addcaster <casterName> or !addcaster <casterName> <customMessage>
RESPOSTA:
A resposta dependerá da maneira de usar o comando. O log de uso desse comando é rastreado pelo script e a resposta com falha ou êxito será enviada ao whisper do usuário.
IMPORTANTE:
  • Se nada for escrito depois de <casterName>, o script adicionará uma entrada para <casterName*gt; no seu arquivo de casters para fazer ShoutOuts usando a Default Message definida em c.3
  • TUDO o que vier depois do <casterName> será interpretado pelo script como sua mensagem personalizada e o script tentrará definir como a mensagem personalizada. Com isso em mente, lembre-se: ESTA MENSAGEM PERSONALIZADA PRECISA seguir as regras - e exemplo - já detalhada sobre como definir uma mensagem em c.3
d.2) Command to edit information from a caster to AutoShoutOut:
Com esse comando, você e seus editores podem alterar as informações salvas dentro do seu arquivo de casters. Esses são os detalhes que podem ser editados:
  • name: troca o nome (também conhecido como Twitch handle/username) de um caster salvo em seu arquivo de casters
  • msg: troca a mensagem usada para ShoutOuts de um caster em seu arquivo de casters
  • allow: troca o comportamento referente a ShoutOuts direcionados a um caster salvo em seu arquivo de casters
USO:
  • Trocando o nome de um caster:
    • !setcaster <casterName> name <newCasterName>

  • Trocando a mensagem de ShoutOut de um caster de "Default Message" para uma mensagem customizada:
    • !setcaster <casterName> msg <customMessage>
    (<customMessage> DEVE seguir as regras especificadas em c.3)

  • Trocando a mensage customizada de um caster para "Default Message" (definda em c.3):
    • !setcaster <casterName> msg Default
    (Default DEVE ser escrito com "D" maiúsculo)

  • Trocando o comportamento do bot aos ShoutOuts:
    • !setcaster <casterName> allow yes (bot enviará ShoutOut para <casterName> utilizando as configurações já definidas)
    !setcaster <casterName> allow no (bot não enviará ShoutOut para <casterName>)
RESPOSTA:
A resposta fornecida a este comando dependerá de que alteração está sendo solicitada e se essa alteração foi feita com sucesso ou não. O resultado da (tentativa de) alteração será SEMPRE enviado ao whisper do usuário.
d.3) Command to delete a caster from AutoShoutOut:
Este comando é usado por você e seus editores para excluir as informações referentes a um único caster (através de seu nome) de dentro do arquivo de casters.
USO:
!delcaster <casterName>
RESPOSTA:
O script procurará a entrada referente a <casterName> no arquivo de casters e a excluirá do arquivo. Se <casterName> não for encontrado, nenhuma ação é tomada. A resposta enviada ao whisper do usuário depende do sucesso ou falha da remoção de <casterName> do arquivo de casters.
d.4) Command to check info about a saved caster:
Este comando é o único neste grupo usado por moderadores, editores e você para inspecionar um conjunto de informações básicas sobre um único caster salvo no arquivo de casters.
USO:
  • Inspecionando todas as informações básicas:
    • !chkcaster <casterName> all

  • Inspecionando a mensagem a ser postada em ShoutOuts:
    • !chkcaster <casterName> msg

  • Inspecionando o comportamento do bot referente aos ShoutOuts:
    • !chkcaster <casterName> allow

    RESPOSTA:
    A resposta dependerá do que está sendo inspecionado assim como dependerá também se a solicitação retornou um resultado bem-sucedido ou não relacionados a <casterName>.
    d.5) List all saved Casters
    Clicar neste botão imprimirá na seção Log do ChatBot uma lista com todo o conteúdo do arquivo de casters.

    d.6) I am sure I want to reset my casters file.
    Selecionar essa opção é OBRIGATÓRIA APENAS se você deseja redefinir o arquivo de casters para o status inicial, sem casters salvos.

    d.7) I want to create a backup of current casters file. (Recomended)
    Habilitar isso é OPCIONAL. Se essa opção estiver ativada, o script copiará todas as informações do último arquivo de caster salvo em um arquivo com a extensão *.bk ("bk" significa backup), antes de apagar todos os dados do arquivo principal de casters.
    NOTA: Se já houver um arquivo *.bk no diretório do script, marcar esta opção substituirá o último backup salvo (e existente) de casters, substituindo todos os dados existentes ali.
    d.8) Reset Casters File
    Isso é auto-explicativo e funcionará corretamente SOMENTE se d.6 estiver ativado.

    d.9) Revert casters file
    Isso carregará o arquivo de casters de volta ao estado anterior, com base no último backup criado (caso exista algum).

    e) ScriptLog Settings:


    Ver em tamanho real (nova aba)

    Este é outro grupo auto-explicativo. Essas configurações estão aqui para gerenciar os LogFiles de uso de script sobre quando/como ele foi usado durante uma sessão de stream. Para usuários que gostam de ter mais controle sobre o que está acontecendo ou desejam relatar algum bug no script, essas configurações do LogFile estão aqui para esse fim.

    e.1) Display Script log in the ChatBot?
    Por padrão, esse recurso está desativado e, como o proprio nome diz, exibirá o log de uso "ao vivo" na seção de log do ChatBot ("Scripts > Logs").

    e.2) AutoSave ScriptLog file?
    Por padrão, esse recurso está ativado. Isso garantirá que um arquivo de log seja salvo no diretório Log criado pelo script quando:
    • o bot for fechado
    • o script é recarregado dentro do bot
      • ou
    • o script for desativado no bot
    e.3) Save ScriptLog File
    Clicar aqui salvará prematuramente os logs de script no LogFile e impedirá que o recurso AutoSave (se ativado) crie um (novo) arquivo de log.
    NOTA: Clicar neste botão também impedirá o salvamento de outro arquivo ou a reescrita do conteúdo já gravado com outras informações de log.
    e.4) Open ScriptLog Directory
    Isso abrirá o diretório em que todos os script LogFiles estão armazenados.

    IMPORTANTE:

    Habilite o recurso "Display Scriptlog in the chatbot?" SOMENTE SE você quiser executar a depuração para BugRerpot (ou por razões de desenvolvimento). Caso contrário, seu ChatBot pode ser sobrecarregado com mensagens de depuração.
