Multi-Streaming Utility

Guia Rápido e informações extras sobre o script

Esse script foi desenvolvido inicialmente para me ajudar (como streamer) a gerenciar facilmente minhas sessões de multi-streaming. Durante meses, passei por streamers fazendo sessões de multi-streaming e principalmente tendo que adicionar esse recurso como um comando independente aos seus bots, "forçando-os" a editar o comando toda vez que quisessem usar com diferentes amigos. Então, pensando nessas streamers (e, é claro, em mim), eu vim com esse script para o chatbot independente do StreamLabs.
Ainda não sei se vou trabalhar em uma versão para o mesmo script para o bot embutido no StreamLabs: OBS, mas, no momento, considerando a capacidade de desenvolver coisas para esse bot independente, manterei apenas essa versão.
É desnecessário dizer, mas farei esta observação aqui: caso você não possua a versão autônoma do bot, é possível baixá-lo aqui. Também é importante observar que (até o momento) os recursos dos serviços de multi-streaming disponíveis aqui são apenas para o Twitch.

Abaixo, descreverei alguns dos recursos que este script realmente fornece:
  • Configuração fácil de comandos
  • Envio de mensagens automáticas com link de serviço Multi-Stream, definida por um timer
  • Configurações do jogo nas configurações de script ou via comando
  • Criação de arquivo de log com o uso do script durante uma sessão de stream
Agora é hora de analisar profundamente esse script.

IMPORTANTE:

Quaisquer alterações feitas em QUAISQUER dos campos neste script serão efetivadas apenas após o botão "Save Settings" (no final da interface do usuário do script) for pressionado.

General Settings:


Ver em tamanho real (nova aba)

Neste grupo, você gerencia as configurações básicas do script. Coisas como a mensagem que você deseja exibir no bate-papo, o provedor de serviços de transmissão múltipla e o jogo que você jogará com seus amigos. Você também pode deixar o campo "Jogo" vazio. Isso forçará o script a selecionar automaticamente o jogo que você definir no diretório no seu Twitch Dashboard ".

1. Message to send on chat:
Este campo é usado para definir a mensagem que o bot exibirá no bate-papo sempre que o comando !multi for usado ou pela mensagem automática enviada pelo script. Alguns parâmetros são necessários aqui para que isso funcione corretamente:
  • {caster}: mostra o nome do streamer
  • {friends}: mostra os nomes dos amigos com quem o streamer irá fazer a transmissão múltipla
  • {link}: mostra o link com a site onde a tranmissão múltipla está sendo feita
  • {game}: mostra o jogo que está sendo jogado com {friends}
2. Site for multi-streaming:
Este é um campo seletor para gerenciar qual serviço de streaming múltiplo será usado para o link criado pelo script. Neste momento, três serviços possíveis estão configurados, mas outros poderão vir em futuras atualizações.

3. Game to be played:
Este campo exibirá o jogo que o usuário configurou. Caso o campo esteja vazio, o script definirá o jogo baseaso pela informação do painel do streamer na Twitch. Também é possível mudar o jogo via comando, como será explicado mais adiante.

Command Settings:


Ver em tamanho real (nova aba)

Este grupo gerencia todas as entradas manuais necessárias para o script funcionar. Em outras palavras, esses campos são para definir os comandos que você deve usar no bate-papo para alimentar parâmetros para o script, fazer ajustes e tudo o mais que é administrável via entrada de bate-papo. A seguir, vemos exemplos de uso do script, usando as configurações de comando padrão e mostrando respostas ilustrativas:

1. Command to set multi-stream link:
Por padrão, o comando é definido como !setmultion. Este comando definirá o link e iniciará o contador para a exibição da mensagem automática, caso o recurso de mensagem da exibição automática esteja ativado. O usuário pode alterá-lo para o que quiser. É importante que ESSE COMMANDO é usado usado apenas por Moderadores, Editores e pelo próprio Streamer.
Comando: !setmultion friend1 friend2 friend3
Resposta: Multi-stream link is set correctly. Viewers can now type !multi in chat to display this link: http://provider/caster/friend1/friend2/friend3/ where you are playing game.
Se usado corretamente, o usuário receberá um sussurro do bot dizendo que o link foi criado e exibirá uma prévia do link para fins de "teste". Caso contrário, o sussurro enviado ao usuário dirá que o comando estava sem os argumentos necessários para criar o link.

2. Command to reset the multi-stream link:
De maneira semelhante à !setmultion, este comando aqui faz exatamente o oposto. Ao usar !setmultioff, você desativará o link da URL de transmissão múltipla, removerá a lista de amigos e cancelará a exibição de mensagens automáticas no bate-papo. O usuário receberá um sussurro do bot, dizendo que a lista de amigos está vazia, portanto, nenhum link de multitransmissão está definido.
Comando: !setmultioff
Resposta: Multi-stream is disabled. All friends erased...
3. Command to set game manually:
Este comando permitirá que o usuário altere o jogo que será exibido junto com o URL no chat da stream.
Comando: !setmultigame GamePlayed
Resposta: Game was updated. Viewers can now type !multi in chat to display this link: http://provider/caster/friend1/friend2/friend3/ where you are playing GamePlayed.
O usuário também pode tentar usar auto após o comando !setmultigame. Ao usar isso, o script obterá o jogo que o streamer está atualmente jogando (ou jogou em sua última sessão de stream) na Twitch, baseado nas informações da Dashboard do streamer.

4. Command to send multi-stream link on chat and User coooldown (in minutes):
Esse é o comando que os usuários precisarão digitar no chat para exibir o link de transmissão multipla no chat. Este comando possui um cooldown definido pelo campo "User cooldown (im minutes):", que por padrão é definido como 5 (cinco) minutos.
Comando: !multi
Resposta: caster is currently multi-streaming with friend1/friend2/friend3 at http://provider/caster/friend1/friend2/friend3/ ! Click the link to a multi-view of GamePlayed!

AutoMessage Settings:


Ver em tamanho real (nova aba)

Este grupo é para gerenciar como o script postará mensagens automáticas no bate-papo, durante o fluxo. Ambos os campos são auto-explicativos; portanto, não há muito o que detalhar nesta seção.

1. Allow to send auto multi-stream link?
Por padrão, esse recurso está ativado e postará a seguinte mensagem no bate-papo com base no cooldown definido pelo campo "Cooldown for auto Multi-Stream link post in chat (in minutes):". Isso postará a mesma mensagem definida conforme indicado acima, por exemplo:
Resposta: caster is currently multi-streaming with friend1/friend2/friend3 at http://provider/caster/friend1/friend2/friend3/ ! Click the link to a multi-view of GamePlayed!

NOTA: O cooldown para envio de mensagens automáticas em chat (por padrão) está definido para 15 (quinze) minutos.

ScriptLog Settings:


Ver em tamanho real (nova aba)

Este é outro grupo auto-explicativo. Essas configurações estão aqui para gerenciar os LogFiles de uso de script de quando/como ele foi usado durante uma sessão de stream. Para usuários que gostam de ter mais controle sobre o que está acontecendo ou desejam relatar algum bug no script, essas configurações do LogFile estão aqui para essa finalidade.

1. Display Script log in the ChatBot?
Por padrão, esse recurso está desativado e, quando ativado, exibirá o log de uso "ao vivo" na seção "Scripts>Logs" do ChatBot.

2. AutoSave ScriptLog file?
Por padrão, esse recurso está ativado. Isso fará com que um arquivo de log seja salvo no diretório Log criado pelo script quando:
  • o bot é fechado
  • o script é recarregado dentro do bot
  • o script é desativado dentro do Bot
3. Save ScriptLog File
Este botão é usado para salvar manualmente o arquivo de log antes de você fechar o ChatBot.

4. Open ScriptLog Directory
Isso abrirá o diretório em que todos os ScriptLogs estão armazenados.

4.1. ScriptLog file example
Você pode verificar um exemplo de um arquivo ScriptLog aqui.

IMPORTANTE:

Ative o recurso "Display Scriptlog in the chatbot?" apenas SE você quiser executar uma depuração (ou por motivos de desenvolvimento). Caso contrário, o seu ChatBot pode ficar sobrecarregado com mensagens de depuração.

Support:


Ver em tamanho real (nova aba)

Esta seção do script é formada apenas por botões, conforme descrito abaixo:

1. Visit my Site
Isso abrirá o navegador padrão e o enviará ao meu site.

2. Open ReadMe.txt
Isso abrirá um arquivo ReadMe.txt com uma versão em texto das informações fornecidas nesta página.

3. Changelog
Isso abrirá o arquivo ChangeLog, onde é possível verificar todas as alterações feitas no script.

4. Save Settings
Este botão, embora esteja fora do grupo Support, é usado para salvar todas as alterações feitas nos campos descritos acima.

  • Detalhes sobre todas as etapas feitas pelo usuário que encontrou o bug
  • Uma cópia do LogFile onde o bug foi detectado
Observe que fornecer todas as informações acima ajudará na correção rápida de erros e fará com que sua requisição seja atendida rapidamente!
