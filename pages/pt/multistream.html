Multi-Streaming Utility

Guia Rápido e informações extras sobre o script

Esse script foi desenvolvido inicialmente para me ajudar (como streamer) a gerenciar facilmente minhas sessões de Multi-Streaming. Durante meses, passei por streamers fazendo sessões de Multi-Streaming e principalmente tendo que adicionar esse recurso como um comando independente aos seus bots, "forçando-os" a editar o comando toda vez que quisessem usar com diferentes amigos. Então, pensando nesses streamers (e, é claro, em mim), eu vim com esse script para o chatbot independente do StreamLabs.
Ainda não sei se vou trabalhar em uma versão para o mesmo script para o bot embutido no StreamLabs: OBS, mas, no momento, considerando a capacidade de desenvolver coisas para esse bot independente, manterei apenas essa versão.
É desnecessário dizer, mas farei esta observação aqui: caso você não possua a versão autônoma do bot, é possível baixá-lo aqui. Também é importante observar que (até o momento) os recursos dos serviços de Multi-Streaming disponíveis aqui são apenas para o Twitch.

Abaixo, descreverei alguns dos recursos que este script realmente fornece:
  • Configuração fácil de comandos
  • Envio de mensagens automáticas com link de serviço Multi-Stream, definida por um timer
  • Configurações do jogo nas configurações de script ou via comando
  • Criação de arquivo de log com o uso do script durante uma sessão de stream
Agora é hora de analisar profundamente esse script.

IMPORTANTE:

A) Quaisquer alterações feitas nas informações fornecidas na interface do usuário do script entrarão em vigor somente após o botão "Save Settings" for pressionado. Para garantir que as alterações foram salvas, é recomendável pressioná-lo duas vezes e sempre recarregar o script após salvar configurações.
B) Lembre-se de que as alterações feitas usando comandos no chat não serão incluídas na interface do usuário do script como novas configurações para o script.
C) Todos os exemplos fornecidos aqui são baseados em valores padrão para campos na interface do usuário do script.
D) É importante que o caster, editores e moderadores do canal que usarão esse script tenham a conta bot do caster adicionado à lista de amigos, porque esse script enviará a maioria das respostas dos comandos em seu whisper, caso os mencionados tenham seu sussurro definido para não receber whisper de não-amigos.

a) General Settings:


Ver em tamanho real (nova aba)

Esse grupo é responsável pelas configurações gerais do script que serão controladas pelos comandos (listados e detalhados mais adiante neste documento). Mensagens, "host" de Multi-Stream, jogo e amigos com os quais você fará a sessão de Multi-Stream podem ser definidos aqui. Abaixo, confira os detalhes em cada campo deste grupo:

a.1) Message to send on chat:
Este campo é usado para definir a mensagem que o bot exibirá no chat sempre que o comando específico para exibir o link for usado ou pela mensagem automática enviada pelo script. Esses são os parâmetros necessários para este campo:
  • {caster}: isso será substituído pelo nome do streamer
  • {friends}: isso será substituído pelos amigos com os quais o streamer está fazendo a sessão de Multi-Stream
  • {link}: isso será substituído pela URL de Multi-Stream criada pelo script
  • {game}: isso será substituído pelo jogo que está sendo jogado com {friends}
Examplo de mensagem aceita pelo script:
{caster} is playing {game} with {friends} Go check them playing together at {link} !
IMPORTANTE:
É ALTAMENTE RECOMENDÁVEL colocar um espaço em branco depois de {link} para evitar a criação de "links quebrados".
a.2) Friends you are streaming with:
Este campo conterá a lista de casters que o script precisa processar para criar a URL de Multi-Stream. É importante notar algumas coisas sobre esse campo:
  • O script criará automaticamente um link (embora não permita exibi-lo ainda no bate-papo) automaticamente se esse campo não estiver vazio.
  • Se este campo estiver vazio, você precisará fornecer uma lista de casters/friends usando o comando específico (a ser detalhado mais adiante) esta documentação.)
  • A lista de casters deve ser separada por espaços em branco, como indicado abaixo:
    • caster1 caster2 caster3
a.3) Site for Multi-Streaming:
Este é um campo seletor onde o usuário pode escolher seu "host" de Multi-Stream favorito. É importante observar que agora temos ( id = x ) (onde x é um número) ao lado do nome do site selecionado. Esse valor de id (número) pode ser usado para alterar/atualizar o link usando um determinado comando (que será explicado mais adiante neste documento).

a.4) Game to play with friends:
Este campo é usado para configurar o jogo que será jogado durante a sessão de Multi-Stream. O campo também pode ser deixado em branco ou ser preenchido com auto. Ao deixá-lo em branco ou preenchê-lo com auto, o script fará com que o jogo a ser mostrado na mensagem seja coletado automaticamente da dashboard do streamer. Também é possível mudar o jogo via comando, como será explicado mais adiante.

b) Command Settings:


Ver em tamanho real (nova aba)

O usuário pode personalizar os comandos usados no bate-papo por este script neste grupo. Este grupo é composto por:

b.1) Command to set/enable Multi-Stream link:
Este comando definirá o link Multi-Stream com as configurações definidas em a e iniciará o timer para a exibição da mensagem automática (caso o recurso de mensagem da exibição automática esteja ativado). O usuário pode alterá-lo para o que quiser e é importante observar aqui que ESTE comando pode ser usado SOMENTE pelo caster, editores e moderadores do canal.

b.2) Command to disable the Multi-Stream link:
Este comando funciona de maneira semelhante ao anterior, mas sua principal função é desativar o link Multi-Stream, desativando também a exibição automática do link no chat (considerando que o recurso de exibição automática está ativado). ESTE comando pode ser usado SOMENTE pelo caster, editores e moderadores do canal.

b.3) Command to set game manually:
Este comando permite ao usuário a possibilidade de atualizar o jogo para ser exibido na mensagem enviada no chat quando o link Multi-Stream é exibido. Este comando pode ser usado SOMENTE pelo caster, editores e moderadores do canal.

b.4)Command to set site manually:
Este comando é usado para atualizar o link Multi-Stream exibido, alterando o "host" do Multi-Stream. Este comando pode ser usado SOMENTE pelo caster, editores e moderadores do canal.

b.5) Command to send Multi-Stream link on chat and User coooldown (in minutes):
Este é o comando usado por todos no canal para exibir o link Multi-Stream criado por este script no chat.

b.6) User cooldown (in minutes):
Esse slider serve para definir o tempo de cooldown por usuário (do comando definido em b.5), com o intuito de evitar spam no link no chat. O cooldown mínimo permitido é de 1 minuto, enquanto o máximo permitido é de 15 minutos.
NOTAS:
a. Casters, editores, and moderadores não são afetados pelo cooldown de usuário definido em b.6.
b. Para impedir spam do comando por estes, um pequeno cooldown de 10 segundos foi adicionado ao comando.
Comandos do script em ação:
  • Ajustando e/ou habilitando o link de Multi-Stream:
    • Usando os valores definidos na interface de usuário (item a.2):
      • !setmultion
    • Usando valores customizados ou atualizando o link já criado:
      • !setmultion <casters>
      (<casters> é a lista de casters a ser mostrada junto com o link, separados por um espaço em branco)
    RESPOSTA:
    A resposta para esse comando dependerá do sucesso ou falha na criação do link para Multi-Stream e será enviada ao whisper do usário.

  • Desabilitando o link Multi-Stream:
    • !setmultioff
    RESPOSTA:
    O bot enviará um whisper ao usuário informando que o ink Multi-Stream foi desabilitado.

  • Atualizando o game a ser mostrado na mensagem de chat:
    • Obtendo o jogo definido pelo caster na dashboard Twitch:
      • !setmultigame or !setmultigame auto
      (auto DEVE ser escrito somente com letras minúsculas para o perfeito funcionamento)
    • Usando um nome customizado:
      • !setmultigame <game>
      (tudo o que for escrito depois de <game> é definido como o nome do jogo, assim o usuário nao precisa se preocupar com jogos que tenham nomes extensos)
    RESPOSTA:
    A resposta enviada ao whisper do usuário depende da forma como o comando foi usado.

  • Atualizando o "host" de Multi-Stream:
    • !setmultisite <id>
    (<id>, como já mencioando em a.3, é a referência numérica próxima ao site de Multi-Stream mostrada na interface de usuário)
    RESPOSTA:
    A resposta dependerá da id utilizada. Uma mensagem de sucesso ou falha na troca será enviada ao whisper do usuário.

  • Mostrando o link Multi-Stream no chat:
    • !multi
    RESPOSTA:
    A resposta depende do link criado. O script também possui uma mensagem padrão para ser mostrada caso o link não tenha sido criado corretamente ou não esteja ativo. O script também prevê os casos relacionados a cooldown e envia ao usuário uma mensagem informando se o usuário (ou o comando) está sob efeito de cooldown e por quanto tempo esse efeito ainda está ativo.

c) AutoMessage Settings:


Ver em tamanho real (nova aba)

Este grupo é para gerenciar como o script postará mensagens automáticas no bate-papo, durante a stream. Ambos os campos são auto-explicativos, portanto, não há muito o que detalhar nesta seção.

c.1) Allow to send auto Multi-Stream link?
Essa é uma função de timer interna para esse script. Com isso, o caster pode permitir (ou não) o script enviar o link Multi-Stream no bate-papo automaticamente, usando um cooldown definido para esse "timer", sem ter que se preocupar em criar um timer específico para exibir o link criado.

Cooldown for auto Multi-Stream link post in chat (in minutes):
Esse slider é usado para definir o tempo entre as postagens de mensagens automáticas pelo timer interno. Os valores mínimo e máximo para esse tempo são 0 e 60 minutos, respectivamente.

d) ScriptLog Settings:


Ver em tamanho real (nova aba)

Este é outro grupo auto-explicativo. Essas configurações estão aqui para gerenciar os LogFiles de uso de script de quando/como ele foi usado durante uma sessão de stream. Para usuários que gostam de ter mais controle sobre o que está acontecendo ou desejam relatar algum bug no script, essas configurações do LogFile estão aqui para essa finalidade.

d.1) Display Script log in the ChatBot?
Por padrão, esse recurso está desativado e, quando ativado, exibirá o log de uso "ao vivo" na seção "Scripts>Logs" do ChatBot.

d.2) AutoSave ScriptLog file?
Por padrão, esse recurso está ativado. Isso fará com que um arquivo de log seja salvo no diretório Log criado pelo script quando:
  • o bot é fechado
  • o script é recarregado dentro do bot
  • o script é desativado dentro do Bot
d.3) Save ScriptLog File
Este botão é usado para salvar manualmente o arquivo de log antes de você fechar o ChatBot.

d.4) Open ScriptLog Directory
Isso abrirá o diretório em que todos os ScriptLogs estão armazenados. Você pode verificar um exemplo de um arquivo ScriptLog aqui.

IMPORTANTE:

Ative o recurso "Display Scriptlog in the chatbot?" apenas SE você quiser executar uma depuração (ou por motivos de desenvolvimento). Caso contrário, o seu ChatBot pode ficar sobrecarregado com mensagens de depuração.

e) Support:


Ver em tamanho real (nova aba)

Esta seção do script é formada apenas por botões, conforme descrito abaixo:

e.1) Visit my Site
Isso abrirá o navegador padrão e o enviará ao meu site.

e.2) Open ReadMe.txt
Isso abrirá um arquivo ReadMe.txt com uma versão em texto das informações fornecidas nesta página.

e.3) Changelog
Isso abrirá o arquivo ChangeLog, onde é possível verificar todas as alterações feitas no script.

e.4) Save Settings
Este botão, embora esteja fora do grupo Support, é usado para salvar todas as alterações feitas nos campos descritos acima.
