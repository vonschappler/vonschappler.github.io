Multi-Streaming Utility

Quick reference and extra info about this script

This script was initially developed to help me (as a streamer) to manage my Multi-Streaming sessions easily. For months I crossed around streamers making Multi-Streaming sessions and mostly having to add this feature as a stand-alone command to their bots, "forcing" them to edit the command (or create a new one) every time they wanted to use with different friends. So, thinking on those streamers (and, of course, on me) I came with this script for the StreamLabs stand-alone chatbot.
I still don't know IF I will work on a version for the same script for the in-built StreamLabs: OBS bot, but for the moment, considering the ability to develop things for this stand-alone bot, I will keep only with this version.
It is needless to say, but I will make this note here: if you don't have the stand-alone version of the bot, you can download it from their page here. It's also important to note that (so far) the Multi-Streaming hosts resources available here are for Twitch only.

Below I will describe some of the features this script actually provides:
  • Easy commands configuration
  • Auto-message with Multi-Stream host link, defined by a timer
  • Game settings within the script settings or via command
  • Log file output with the script usage during a specifc stream session
It's now time to take a deep look at this script.

IMPORTANT:

A) Any changes made to the information provided in the script UI will take effect only after "Save Settings" button is pressed. To ensure the changes were saved, it's advised to press it two times and always reload the script after saving settings.
B) Keep in mind that changes made using commands in chat won't be cased on the script UI as new settings to the script.
C) All examples given here are based on default values for fields in the script UI.
D) It's important that caster, editors, and moderators of the channel that will use this script have the caster's bot account added to their friends' list, because this script will send most of the answers of commands to their whisper, in case those mentioned have their whisper set to don't receive whispers from non-friends.

a) General Settings:


View image in real size (new tab)

This group is responsible for the general settings from the script which will be controlled by the commands (listed and detailed later in this document). Messages, Multi-Stream "host", game and casters you are Multi-Streaming with can be set here. Below, there are details on each field of this group:

a.1) Message to send on chat:
This field is used to define the message the bot will display in chat whenever the specific command to display the link is used or by the auto-message send by the script. Those are the required parameters for this field:
  • {caster}: this will be replaced by the caster name
  • {friends}: this will be replaced by the friends the streamer is streaming with
  • {link}: this will be replaced by the Multi-Streaming url created by the script
  • {game}: this will be replaced by the game being played with {friends}
Example of working message:
{caster} is playing {game} with {friends} Go check them playing together at {link} !
IMPORTANT:
It's STRONGLY ADVISED to add a blank space after {link} to avoid a broken link creation.
a.2) Friends you are streaming with:
This field will hold the list of casters the script needs to process in order to create the Multi-Stream URL. It's important to note a few things about this:
  • The script will automatically create a link (although it won't enable displaying it on chat) automatically if this field is not empty.
  • If the this field is empty, you need to give a list of casters/friends using the specific command (to be detailed later in this documentation.)
  • Names inside this field must be separated by a blank space as shown below:
    • caster1 caster2 caster3
a.3) Site for Multi-Streaming:
This is a dropdown field where the user can select his favorite Multi-Streaming "host". It's important to notice the ( id = x ) (where x is a number) next to the name of the selected "host". This id value (number) can be used to change/update the link using a specific command (which will be explained later in this document).

a.4) Game to play with friends:
This field is used to set up the game which is going to be played during the Multi-Stream session. The field also can be left blank or be filled with auto. By leaving it blank or filling it with auto will make the script to get the game played from the caster's Twitch dashboard. It's also possible to change the game via command, as it will be explained later.

b) Command Settings:


View image in real size (new tab)

The user can customize the commands used in chat by this script in this group. This group is composed of:

b.1) Command to set/enable Multi-Stream link:
This command will both set the Multi-Stream link defined with the settings defined on a and start the counter for the auto-message display (in case the auto-display message feature is enabled). The user can change it to anything they like and it's important to note here that THIS command can be used ONLY by the caster, editors, and moderators of the channel.

b.2) Command to reset the Multi-Stream link:
This command works in a similar way to the previous one, but its main function is to disable the Multi-Stream link, turning off the auto-display in chat (considering the auto-display feature is enabled). THIS command can be used ONLY by the caster, editors, and moderators of the channel.

b.3) Command to set game manually:
This command allows the user the possibility of updating the game to be displayed in the message sent on chat when the Multi-Stream link is displayed. THIS command can be used ONLY by the caster, editors, and moderators of the channel.

b.4) Command to set site manually:
This command is used to update the Multi-Stream link displayed, changing the Multi-Stream "host". THIS command can be used ONLY by the caster, editors, and moderators of the channel.

b.5) Command to send Multi-Stream link on chat:
This is the command used by everyone in the channel in order to display the Multi-Stream link created by this script in chat.

b.6) User cooldown (in minutes):
This slider is to set user's cooldown of the command defined in b.5 to avoid spamming the link in chat. The minimum cooldown allowed is 1 minute while the maximum allowed by this is 15 minutes.
NOTES:
a. Casters, editors, and moderators won't be affected by the user cooldown defined on b.6.
b. To refrain the permission levels above from "quick" spamming the link it was added a small cooldown of 10 seconds.
Script commands in action:
  • Setting and/or enabling Multi-Stream link:
    • Using values saved on script UI (item a.2):
      • !setmultion
    • Using custom values or updating a previously created link:
      • !setmultion <casters>
      (<casters> is a list of casters to be displayed, separated by a blank space)
    ANSWER:
    The answer for this command will depend on the success or fail of Multi-Stream link creation and it will be sent on the user's whisper.

  • Disabling Multi-Stream link:
    • !setmultioff
    ANSWER:
    The bot will send a whisper to the user to inform the Multi-Stream link was disabled.

  • Updating the game to be displayed in the Multi-Stream chat message:
    • Retrieving game name from Twitch's caster dashboard:
      • !setmultigame or !setmultigame auto
      (auto MUST be written with no capital letters in order for this to work)
    • Using a new/custom game name:
      • !setmultigame <game>
      (everything written after the command will be defined as <game>, so the user don't need to worry about games with "big" names)
    ANSWER:
    The answer for this command will be sent to the user whisper and it will depend on the user's input for this command.

  • Updating the Multi-Stream "host":
    • !setmultisite <id>
    (<id>, as already mentioned in a.3, is the number reference to the Multi-Stream site to be used, displayed in the dropdown selector on the script UI)
    ANSWER:
    The answer depends on the id entered. A success or failure message will be sent to the command user's whisper.

  • Displaying the Multi-Stream link created in chat:
    • !multi
    ANSWER:
    The answer depends on the Multi-Stream link created. The script also has a default message to be displayed in case no link was set up, due to a syntax error on using any of the commands above or because the command to disable the link was used. If the user of the command is under User Cooldown or the command itself is under cooldown, a whisper is sent to the user informing the command is on cooldown and how many minutes until it can be used again.

c) AutoMessage Settings:


View image in real size (new tab)

This group is to manage how the script will post auto-messages in chat, during the stream. Both fields are self-explanatory, so there is not much to detail for this section.

c.1) Allow to send auto Multi-Stream link?
This is a built-in timer function for this script. With this, the caster can allow (or not) the script to send the Multi-Stream link on chat automatically, using a defined cooldown for this "timer", without having to worry in creating a specific timer for displaying the created link.

c.2) Cooldown for auto Multi-Stream link post in chat (in minutes):
This slider is to define the time delay between auto message posts by the timer. The delay goes from 0 to 60 minutes.

d) ScriptLog Settings:


View image in real size (new tab)

This is another self-explanatory group. These settings are here to manage the script usage LogFiles of when/how it was used during a stream session. For users who like to have more control over what's happening or want to report any bug on the script, these LogFile settings are here for that purpose.

d.1) Display Script log in the ChatBot?
By default, this feature is disabled, and as it says it will display the usage log "live" within the ChatBot log section ("Scripts>Logs").

d.2) AutoSave ScriptLog file?
By default, this feature is enabled. This will ensure that a log file is saved under the Log directory created by the script when:
  • the bot is closed
  • the script is reloaded within the bot or
  • the script is disabled within the bot
d.3) Save ScriptLog File
This button is used to manually save the log file before you close the ChatBot.

d.4) Open ScriptLog Directory
This will open the directory where all the ScriptLogs are stored. As reference, it's possible to preview an example of a ScriptLog file here.

IMPORTANT:

Enable the "Display Scriptlog in the chatbot?" feature ONLY IF you want to perform debugging (or for development reasons). Otherwise, your ChatBot may be overloaded with debug messages.
