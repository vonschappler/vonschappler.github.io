Custom ShoutOut (text version)

Quick reference and extra info about this script

Streamers have plenty of ways to support or show appreciation to other streamers. Raids, Hosts, Ads (also called ShoutOuts) are some of those ways. This script basically helps streamers in the Ad "portion" of this support to others.
Also, it's not always that a streamer will have modeators at hand to help him/her in this advertising to fellow streamers, and thinking on this specific situation, I come here to offer you a brand new script that will help with that! The list below resumes what Custom ShoutOut can do for you (as streamer):
  • Create a file with "custom messages" per streamer you want to call out on your stream
  • Allow Auto-ShoutOuts for raiders and saved streamers when they join your channel or send their first message during a stream session (saved streamers only)
  • Offer total control of the saved streamers file, based on commands and UI interface buttons
  • Deep log system to allow control of everything done by the script during a stream session
Let's now go a bit deeper into the specifications of each field of the script!

IMPORTANT:

Any changes made to ANY fields in this script will just take effect after the button "Save Settings" (at the end of the script UI) is pressed and the script is reloaded. It's also necessary to say that most of the replies from the script are sent to the user's whisper, meaning that for the user to make sure the changes made were successful or not, they will need to have your bot added as a friend in order to get all the whispers from it, in case the user has whispering feature blocked on Twitch.

a) Main Settings:


View image in real size (new tab)

This group is responsible for the basic settings of the script. It's where you control the basic behavior of the bot.

a.1) Auto ShoutOut saved casters?
This feature will enable or disable the option for the bot to Auto-ShoutOut casters/streamers saved on your casters' file, which creation will be detailed later.

a.2) ShoutOut when caster joins channel?
If enabled, this feature will send a custom ShoutOut (if defined) whenever a saved caster joins your channel. If disabled, it's necessary for the saved caster to send anything on your chat in order for the bot to ShoutOut the saved caster. Saved casters will be ShoutOut only once per session with this feature. For extra ShoutOuts, it's necessary to do it via command.

a.3) Auto ShoutOut raiders?
This feature will Auto-ShoutOut raiders on your channel based on the settings defined in the next group.

b) Shout Raiders Settings:


View image in real size (new tab)

As already stated, this group controls how the bot will behave if a.3 function is enabled. It's even possible to send a test raid to the script (thanks to the script made by "Kruiser8's Raid Notify script". A few changes where made to his code in order to make this raid testing a bit more realistic.

b.1) Message to use for ShoutOut raiders:
This is where you can set a custom message to be sent as an Auto-ShoutOut whenever you are raided by someone. Note that for this to work correctly, a few parameters MUST be added to your custom message:
  • {raider}: this will be replaced automatically by the raider's name
  • {count}: this will be replaced automatically by the number of viewers raiding you
  • {url}: this will be replaced automatically by the raider's channel URL on Twitch
  • {game}: this will be replaced automatically by the last game played by the raider
Example of working message:
{raider} was playing {game} and just brought {count} viewers to our channel! Let's support them visiting {url} !
IMPORTANT:
The script will check for the presence of the parameters listed above when trying to save the custom message for ShoutOuts to raiders. So for the message to be accepted by the script, remember to always give a blank space after each one of the parameters used and make sure to add ALL the parameters on your message!
b.2) Minimum raiders to Auto ShoutOut:
This specifies the number of viewers the raider needs to bring to your channel in order to be Auto-ShoutOut.

b.3) Save raiders to caster file?
If enabled, this feature will add an entry to the saved casters' file for use later by the script.

b.4) Test Raider AutoSO settings:
Clicking this button, the scrip will send a "fake" raid to test the settings defined above. If a.3 is enabled, the script will display on your Twitch chat the custom message defined on b.1 and if b.3 is enabled, it will save (or update) an entry with the raid details to your casters' file.
As already mentioned, this part of the script was made with the help of already existing script ("Raid Notify") made by Kruiser8, with some modifications to the original code for use in this test situation.

c) ShoutOut Settings:


View image in real size (new tab)

This group here is to manage how the bot will control settings for both "manual" (extra) ShoutOuts, Auto-ShoutOuts and even give the possibility to manualy ShoutOut casters which are not saved on your casters' file.

c.1) Command for manual ShoutOut:
This field is used to set the command used for manual ShoutOuts in case a.1 is disabled or if you see fit making extras ShoutOuts. This command will ShoutOut saved casters on your casters' file.
USAGE:
!caster <casterName> or !caster <@casterName>
ANSWER:
The answer depends on the settings saved for this script. In fact, the bot will look for the entry of casterName on your casters' file and will post a message with an ad to casterName's Twitch Channel.
c.2) Permission to use the command:
This is self explanative. Just users with the required permission will be able to use the command set on c.1. By default, Moderators are the basic permit setting for using this command but it's up to the user to define who they want to be able to ShoutOut casters on their channel.

c.3) Default message for ShoutOuts:
Similar to what was detailed on b.1, this field is used to set an ad message to be posted on chat. This one is called as "Default message" for a reason to be detailed later on this guide. For this message to be accepted by the script, it NEEDS to have the following specified parameters:
  • {caster}: this will be replaced automatically by the caster's name (required parameter)
  • {url}: this will be replaced automatically by the caster's Twitch Channel link (required parameter)
  • {game}: this will be replaced automatically by the caster last played game (optional parameter)
Example of working message:
Let's support my friend {caster} visiting their {url} ! There were playing {game} on their last stream!
IMPORTANT:
The script will check for the presence of the parameters listed above when trying to save the custom message for ShoutOuts to casters. So for the message to be accepted by the script, remember to always give a blank space after each one of the parameters used and make sure to add ALL REQUIRED parameters on your message!
c.4) Global cooldown (in seconds):
Specifies a small cooldown to avoid spam of the command in case it's used by two or more permitted users in a short difference time smaller than the value defined. The default setting value is 5 seconds and it can go up to 60 seconds (1 minute).

c.5) Allow manual shout to unsaved casters?
Enabling this feature will grant permission to the script to ShoutOut casters which are not saved on your casters' file by using the command defined in c.1.

d.4) Caster Control Settings:

View image in real size (new tab)

This section here is the core of this script. First of all, here is where you set all the commands necessary to manage what's going to be added to your casters' file that is going to be used by the script.

d.1) Command to add a new caster to AutoShoutOut:
This is the command you and your editors (yes, it's set by default for security reasons to just editor have access to use it) will add a new caster tor your casters' file. It's important to mention that all casters added to your casters' file will ALWAYS be "ShoutOut" by the script (following the specified settings defined in the fields already detailed), unless specified differently with the ALLOW definition (this setting will be explained later in this guide).
USAGE:
!addcaster <casterName> or !addcaster <casterName> <customMessage>
ANSWER:
The answer will depend on the way of the usage of the command. Logging for this command is tracked by the script and the fail or success answer will be sent to the user's whisper.
IMPORTANT:
  • If nothing is written after <casterName>, the script will add an entry to <casterName> to send ShoutOuts using the Default Message set on c.3
  • EVERYTHING that comes after the <casterName> will be interpreted by the script was your custom message and the script will try to set it as the custom message. With this in mind, remember: THIS CUSTOM MESSAGE NEEDS to follow the specified rules - and example - already detailed on how to set a message on c.3
d.2) Command to edit information from a caster to AutoShoutOut:
With this command, you and your editors can change the informations saved inside your casters' file. Those are the details that can be edited:
  • name: changes the name (aka Twitch handle/username) of a saved caster on your casters' file of a saved caster
  • msg: changes the message set for a caster saved on your casters' file
  • allow: changes the behavior of the script to allow the caster to be or not to be shout by this script
USAGE:
  • Changing a caster name:
    • !setcaster <casterName> name <newCasterName>

  • Changing a caster message from the default message to a custom message:
    • !setcaster <casterName> msg <customMessage>
    (<customMessage> MUST follow the rules specified in c.3)

  • Changing a caster message from a custom to "Default Message" (set in c.3):
    • !setcaster <casterName> msg Default
    (Default MUST BE WRITTEN with capital "D")

  • Changing the bot behavior to ShoutOuts:
    • !setcaster <casterName> allow yes (bot will shout out <casterName> using the defined settings)
    !setcaster <casterName> allow no (bot will not shout out <casterName>)
ANSWER:
The answer provided to this command will depend on what is being requested to be changed and on if the change was made successfully or not. The answer for this request will ALWAYS be sent to the user's whisper.
d.3) Command to delete a caster from AutoShoutOut:
This command is used by you and your editors to delete information for a single caster (by his/her name) from your casters' file.
USAGE:
!delcaster <casterName>
ANSWER:
The script will look out for <casterName> entry on your casters' file and delete it from the file. If <casterName> is not found, no action is taken. The answer sent to the user's whisper depends on the success or fail of removal of <casterName> from casters' file.
d.4) Command to check info about a saved caster:
This command is the only one in this group used by moderators, editors and you to inspect a set of basic information about a single saved caster on your casters' file.
USAGE:
  • Inspecting all basic saved information:
    • !chkcaster <casterName> all

  • Inspecting the message to be displayed in chat during ShoutOuts:
    • !chkcaster <casterName> msg

  • Inspecting the script behavior to ShoutOuts:
    • !chkcaster <casterName> allow

    ANSWER:
    The answer will depend on what is being inspected as it will depend if the request returned a successful result or not related to <casterName>.
    d.5) List all saved Casters
    Clicking this button will print in ChatBot Log section a list with all the contents of your saved casters' file.

    d.6) I am sure I want to reset my casters' file.
    Checking this is MANDATORY ONLY IF you want to reset your casters' file to the initial status, with no casters saved in there.

    d.7) I want to create a backup of current casters' file. (Recomended)
    Enabling this is OPTIONAL. If this option is enabled, the script will copy all the information of the last saved casters' file into a file with the extension *.bk ("bk" stands here for backup), before erasing all the data from the main casters' file.
    NOTE: If there is already a *.bk file on the script directory, checking this option will overwrite the last saved (and existent) backup of casters' file, replacing all data in there.
    d.8) Reset casters' file
    This is self-explanatory and will work properly ONLY if d.6 is enabled.

    d.9) Revert casters' file
    This will load your casters' file back to a previous state based on the last saved backup.

    e) ScriptLog Settings:


    View image in real size (new tab)

    This is another self-explanatory group. These settings are here to manage the script usage LogFiles of when/how it was used during a stream session. For users who like to have more control over what's happening or want to report any bug on the script, these LogFile settings are here for that purpose.

    e.1) Display Script log in the ChatBot?
    By default, this feature is disabled, and as it says it will display the usage log "live" within the ChatBot log section ("Scripts > Logs").

    e.2) AutoSave ScriptLog file?
    By default, this feature is enabled. This will ensure that a log file is saved under the Log directory created by the script when:
    • the bot is closed
    • the script is reloaded within the bot
      • or
    • the script is disabled within the bot
    e.3) Save ScriptLog File
    Clicking here will prematurely save the script logs in the LogFile and will prevent the AutoSave feature (if enabled) to create a (new) log file.
    NOTE: Clicking this button will also prevent from saving another file or rewriting the actual LogFile contents with another Log information.
    e.4) Open ScriptLog Directory
    This will open the directory where all the Script LogFiles are stored.

    IMPORTANT:

    Enable the "Display Scriptlog in the chatbot?" feature ONLY IF you want to perform debugging for BugReport (or for development reasons). Otherwise, your ChatBot may be overloaded with debug messages.
